### A Git与GitHub的核心区别
Git是一款**本地版本控制工具**，它会在项目目录下生成`.git`文件夹，记录每一次代码修改的历史，让我们可以随时回退到某个版本、对比变更差异，不需要依赖网络就能管理代码。
而GitHub是基于Git的**远程代码托管平台**，它把本地的Git仓库同步到云端，既可以备份代码，也能让多个开发者在同一个项目上协作，是程序员之间共享代码、参与开源项目的重要载体。

没有学习之前，我不知道Git与GitHub之间的差别，以为它们是同一个东西，现在才明白形象来所，Git是“照相机”，GitHub是“相册”，在Git我们学的是命令与原理，记录代码的每一次修改，创建分支，回退历史，而GitHubs是一个仓库，装东西的，方便团队的协作。

### B Git基础操作与分支管理
Git的核心操作围绕“工作区-暂存区-本地仓库-远程仓库”展开：
- `git clone`：把远程仓库克隆到本地
- `git add`：把修改的文件加入暂存区
- `git commit`：把暂存区的变更提交到本地仓库，生成版本记录
- `git push`：把本地仓库的版本同步到远程仓库
- `git branch` / `git checkout`：创建和切换分支，让不同功能可以并行开发而不影响主代码


刚开始使用Git时，觉得打字母容易打错，还有空格，还有和之前不一样的操作，比如粘贴复制不能用Ctrl+V,但是也明白了Git这个工具的一些便利，比如这个有回溯功能，不用怕出错，有主分支和新分支，以后在新分支出错了也不要怕，因为我们可以在主分支里面删除它，

### C GitHub协作流程与PR提交
在GitHub上协作的标准流程是：
1.  Fork目标仓库到自己的账号下
2.  克隆Fork后的仓库到本地
3.  创建功能分支并完成开发
4.  提交变更并推送到远程分支
5.  在GitHub上发起Pull Request（PR），请求把自己的代码合并到原仓库
6.  等待原仓库维护者审核、评论、合并

这次提交学习感想的实践，让我完整走了一遍从创建分支到发起PR的流程，虽然过程并不顺利，但是学会了协作，这种可以统一开发，真的很方便，可以保证项目代码的质量与一致性。
