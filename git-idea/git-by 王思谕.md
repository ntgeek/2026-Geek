# Git与GitHub的学习

## 学习背景

Git作为目前世界上最先进的分布式版本控制系统，掌握Git后会大大简单化繁琐的代码管理，为接下来的学习奠定基础。GitHub是运行Git的重要平台，不仅能协同开发，也能向大佬学习，锻炼自己的代码能力。

## 学习内容目录

- Git的核心概念
- GitHub的基础操作

## 学习内容

### Git的核心概念

Git就像一个“只进不出”的容器，可以容纳多人一起开发，但当其中某人出现失误时，并不会影响到主分支；且支持多人同时开发，避免交流过程中文件的损害和时间的浪费。个人开发时要建立*仓库*装项目，不能直接对项目进行修改。然后将本地仓库与*远程仓库*连接起来就可以将自己修改的项目提交到远程仓库；在*分支*上操作不会影响主分支，写完再合并可以避免代混乱；*标签*可以让我们快速定位，避免浪费时间。

Git确实节省了很多时间，基础命令也不难，非常适合我们这类新手

### GitHub的基础操作

- Clone：可以把远程仓库的内容保存到本地仓库，在本地进行修改

- Pull Request：向别人的仓库提供项目修改时要向别人提出请求，对方同意后才能接收修改

- Fork：可以将别人的仓库复制到自己的仓库

- Push：将本地的修改同步到远程仓库

- Pull：将远程仓库的最新内容更新到本地仓库

  

GitHub像一个游戏一样，了解的过程中很有趣，还能个性化设计自己的GitHub。
